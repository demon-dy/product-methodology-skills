# 商业闭环总模型

整条链：

```
真需求 → 商业价值 → 用户/市场共识 → 成交/关系 → 资源回流 → 能力系统/变现/分配 → 更强生存优势
```

《真需求》不是"做产品技巧"，是商业世界的生存方法：洞察需求并交付价值，穿越分歧达成共识，获得资源发展自己的生存模式，用求真能力持续校准"我以为"和"事实如此"的差距。

## 四问诊断（总入口，按顺序过）

1. **价值**：是否有交易对手愿意付费或付出资源？（不是你认为有价值，是对方愿意付费）
2. **共识**：对方为什么不行动？分歧在感知、想象、场景、还是利益？
3. **模式**：获得资源后，能否发展能力系统继续生存？拿谁的钱、能不能持续拿、成本可控吗？
4. **求真**：这些判断来自事实反馈，还是自我感动？

## 常见卡点的快速锚点

- 自己很兴奋、用户无感 → 多半是**价值未被感知**或**伪价值**（价值环 + 共识·感知）
- 价值成立、用户却不动 → **共识**问题，别继续劝"多营销"
- 卖得动但一直靠补贴 → **模式**问题（拿谁的钱）
- to B / 内部项目推不动 → **共识·利益相关人**
- "我觉得行业该这样" → **求真·第一性**，先分清应然与实然

## 输出物

需求真实性判断、商业价值假设、共识卡点、模式风险、下一步验证动作。

> 后记边界：产品思维回应用户需求，但企业能长多大还受环境需求、生态位、定价权约束。不要把产品思维误当全部战略。
