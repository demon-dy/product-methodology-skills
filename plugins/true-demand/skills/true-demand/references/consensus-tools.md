# 共识工具

价值成立但市场不动，别只问"用户为什么不懂我"，要问"我和用户之间的分歧是什么"。

## 四类分歧（共识诊断）

1. **感知分歧**：你做了特性，用户没感知，或感知成了负面（无抗猪肉：创始人激动、用户感知弱）。
2. **想象分歧**：你讲画像，用户行动依赖的是人设、愿景、自我故事。
3. **场景分歧**：产品不错，但没落入高频、强感知、可复购的具体场景。
4. **利益分歧**：相关人不受益甚至受损，不会推动。

## KANO：特性 ≠ 属性

产品做出来的是特性，用户感受到的是属性。

| 属性 | 用户反应 | 管理方式 |
|---|---|---|
| 必备 | 没有就不满，有了也未必惊喜 | 必须稳定交付 |
| 期望 | 越好越满意 | 核心卖点，确定性交付 |
| 魅力 | 有了惊喜，没有也接受 | 创造差异和口碑 |
| 无差异 | 无感 | 不过度投入 |
| 反向 | 做了反而讨厌 | 排除或分人群 |

体验管理核心是预期管理。复购用户才是基本盘。

## 用户画像 vs 用户人设

- **画像**：适合功能价值刚需品。逻辑：用户没有什么，所以需要什么。
- **人设**：适合情绪/资产价值弹性需求品。逻辑：用户越擅长、越认同、越想成为某种人，越持续投入。

关键判断：一个人的长项往往就是他的需求；长期没有某物不一定代表需要，也可能是持续回避；弹性需求产品要进入用户的人设故事，而非只满足冷标签（深圳女孩征婚：冷条件无法描绘真实对象）。

## 场景匹配

没有产品场景匹配，就没有产品市场匹配。场景 = 时间 + 空间 + 情境 + 触发行动的上下文。

工作流：①穷举可能进入的生活/工作场景 → ②识别周期性、频率、强感知、复购机会 → ③聚焦一个**黄金场景**用 200% 力气打磨 → ④设计进入场景的动作、渠道、角色、利益链。

判断标准：是否具体到"谁在何时何地为什么用"？是否高频或强感知？是否有更顺手的替代品？能否形成习惯和复购？（校内网放假大巴、Keep 宿舍瑜伽垫、亲子游泳馆切细场景）

## 利益相关人地图

一件事能成是受益者驱动，能做大是受益者多。横轴利益（受损↔获益），纵轴影响力（负面↔正面）。

用于：内部创业立项、to B 销售、新市场启动、危机公关、组织变革。

核心判断：一个人的态度就是他的利益；领导共识不是演讲，是逐个获得关键影响力角色的首肯；利益链不到终端末梢，用户行动不会自然发生（可口可乐入华：许可→媒体→火锅店→促销员的完整利益链）。to B 交付常需附带组织变革方案（AI 客服：客服主管无动力训练 AI）。

## 认知战（让非共识变共识）

新品类/主张/习惯的建立。媒体认知战五要点：信息操纵（选有力量的信息包）、情绪操控、认知引导（重复强化判断框架）、舆论导向（选阵地和背书）、心理操作。近距离说服四词：**符**（找对方内心真实的决定种子）、**权**（运营局面、改变相关人态度）、**决**（让对方做决定和行动）、**枢**（抓变化背后不变的轴心）。

> 伦理边界：认知战只能让真实价值被理解、被感知、被选择。包装伪价值就是流寇战术。

## 关系四层（共识的成果）

| 层级 | 特征 | 商业含义 |
|---|---|---|
| 需要 | 刚性、明确、冷漠 | 工具关系，易替换 |
| 喜欢 | 有倾向性 | 有偏好，无忠诚 |
| 认同 | 开始有忠诚 | 用户视品牌为同类 |
| 归属 | 愿意为之而战 | 愿投入、维护、参与 |

认同是消费者与粉丝的分界，归属是散粉与核心共同体的分界。
