# 科学做事 6 原则

不是产品专属，是做任何需决策工作的思维方式。核心作用：**把"感觉"转成"可检验的判断"**。

1. **先问"我在解决什么问题"**：动手前用一句完整的话写下来——"我们发现 [具体用户] 在 [具体场景] 下有 [具体困扰]，导致 [具体后果]。" 写不出或团队理解不一致 = 还没想清楚。别跳过问题定义直接给方案。

2. **数据说话，不用感觉**：结论必须有数据/可量化观察支撑，不接受"我觉得""大家都说"。"说喜欢"和"真付钱/真留下"是两回事。数据缺失处，设计可量化观察（访谈记录、操作录屏、点击热图）。

3. **假设先行，再找证伪**：先写下你相信什么（越具体越好），再设计最快最低成本的方式去**打倒它**，主动找反例。因为人天然有确认偏误，会去找支持自己的证据。"问用户喜不喜欢我们的想法"几乎必然得到客气的"喜欢"，不带预测力。

4. **分清"观察"和"解释"**：观察 = 用户做了什么、数字多少（客观）；解释 = 你认为他为什么这么做（需数据/访谈支撑）。例：一个用户提交了 81 次申请，可能是"离不开"，也可能是"流程太难用反复出错"——两种解释带来完全相反的决策。别把"用户很积极地用，说明很喜欢"这种混淆当结论。

5. **小步验证，不赌大局**：任何大动作前先问"我能用什么最小代价先验证关键假设？" 想做新功能 → 先放假按钮看有没有人点；想搞地推 → 先 3 人试跑一周看转化再定规模。验证失败损失小，成功再加大投入。

6. **结论要可操作**：不能转成下一步行动的分析等于没分析。写完结论问自己"看了这个，我明天该做什么？" 答案若是"继续观察/深入研究"，说明结论还不够具体。
   - ❌"用户对新手引导满意度不高" → ✅"新手引导第 3 步流失 60%，下周优先重设计第 3 步，目标把该步流失降到 30% 以下"。
