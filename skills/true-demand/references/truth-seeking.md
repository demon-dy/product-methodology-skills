# 求真

## 应然与实然

- **应然**：你认为事情应该如何。
- **实然**：事情实际上如何。

创新者必须在实然世界中看到应然，并用行动把应然变成新的实然。只活在应然 → 空想；只活在实然 → 失去创新。精英常用应然判断、再用数据校验（王小川/搜狗：先应然后实然）；草根常从用户现场长出产品（hao123、李兴平：先实然后应然，极致简单可能就是第一性优势）。弱者离真实用户更近时，可能更接近道。

## 第一性原理

不要只优化当下品类的表象，回到行业/产品存在的根本原因。

训练问题：

- 这个行业为什么存在？
- 用户最古老、最基础、最不变的需求是什么？
- 当前方案是在优化马车，还是在发明汽车？
- 哪些优化与第一性高度相关，哪些只是噪音？

第一性不是现有形态，而是根本目标（马斯克：回到品类根本，具体到零件和责任）。

## 行动力与小闭环

行动力是一个人的自我模式。**拖延的反面是具体**，提高行动力的关键是提升掌控的精确度（火箭第四次成功：清晰过程让失败变成有效反馈）。

真正的下一步不是"想清楚"，而是设计一个能**暴露事实反馈**的小闭环：用最小成本制造一次真实交易/真实使用，观察成败信号，再迭代。失败若产出有效数据，就是迭代成本（特斯拉炸钢获数据）。

### 证据梯子（判断需求是否成立）

越往下越强。低级别证据不能算需求成立：

```
口头说会买 / 问卷打分 / 试吃说好   ← 最弱，几乎不算验证（人嘴上支持，钱包诚实）
点击 / 留资 / 加购 / 排队 / 收藏    ← 行为信号，中等
真实付费 / 预售下单 / 复购          ← 最强，才算需求成立
```

诊断"卖不动/没人买/试吃反馈好却不买单"时，第一步就是把现有证据放上梯子：如果用户只有口头/试吃，需求其实**尚未验证**，别急着归因到"营销不够"。

### 最小验证动作的四件套

合格的验证动作不是"再调研"，而是一次能暴露事实的真实交易，必须写清：

1. **怎么做**：选最轻的真交易形态——预售页 / 假门（fake door）落地页 / concierge（先人工交付，不先建系统）/ 单店单场 / 老客户先卖。
2. **成本与时间盒**：花多少钱、几天内做完。
3. **量化成功阈值**：一个数（如"48 小时 ≥10 人付定金""试点门店周复购 ≥30%"）。
4. **失败信号**：达不到阈值意味着哪个假设被证伪，下一步怎么调。

撤补贴/去掉外力是检验模式与需求的经典动作：把补贴、人情、内部行政推力撤掉，看真实付费是否还在。

> 好产品之美：功能、体验之外还要自然、和谐、恰到好处，能让人一眼感知，不多不少。
